# Security Policy

## Reporting Security Issues

We take the security and privacy of this application very seriously. If you find a security vulnerability, please do **not** report it via public GitHub issues. 

Instead, report it privately by emailing the developer:
- **Hari Moradiya** - harimoradiya123@gmail.com

Please include a detailed description of the vulnerability, steps to reproduce it, and any other relevant information. We will work to investigate and resolve the issue promptly.

## Crucial Reminders
* **Never Publish Credentials or API Keys**: Do not post API keys, keystores, signatures, passwords, or personal credentials in GitHub issues, pull requests, or code files.
* **Keep Local Network Details Private**: Do not post sensitive information (such as local IP addresses of your devices, MAC addresses, or custom TV tokens) in public issues.
* **Remove Logs with Sensitive Data**: Ensure that logs showing private network interactions or pairing payloads are stripped of sensitive details before attaching them to bug reports.
